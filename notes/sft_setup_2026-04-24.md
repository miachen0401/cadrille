## Environment diagnosis that killed the container path

- Hostname `1b0a20681079`, `/proc/1/root` has `NGC-DL-CONTAINER-LICENSE`
- Container's user is `uid=1000 gid=1000`, no `sudo` binary
- `/workspace` is the only bind-mount shared with host; `/usr`, `/etc`, etc.
  are container-local
- User has sudo **on host**, but those apt installs (`libosmesa6-dev`, etc.)
  only reach host `/usr/lib` — invisible from inside container
- Open3D source-build therefore requires a path out of the container

Conclusion: user is switching to **running uv directly on host** so sudo +
apt + build tools all work in one place.

```bash
    "git+https://github.com/CadQuery/cadquery@e99a15df3cf6a88b69101c405326305b5db8ed94"

```

### Step 3 — Source-build Open3D with HEADLESS rendering (30-45 min)
This is what `Dockerfile.official` line 6-17 does. Ship the resulting wheel
into our uv venv.

```bash
git clone https://github.com/isl-org/Open3D.git /tmp/Open3D-build
cd /tmp/Open3D-build
git checkout 8e434558a9b1ecacba7854da7601a07e8bdceb26

mkdir build && cd build
cmake -DENABLE_HEADLESS_RENDERING=ON \
      -DBUILD_GUI=OFF \
      -DUSE_SYSTEM_GLEW=OFF \
      -DUSE_SYSTEM_GLFW=OFF \
      ..
make -j$(nproc)

# install into uv venv
source /path/to/cadrille/.venv/bin/activate  # or: export VIRTUAL_ENV=... ; PATH=$VIRTUAL_ENV/bin:$PATH
make install-pip-package
```

Verify:
```python
import open3d as o3d
print(o3d._build_config['ENABLE_HEADLESS_RENDERING'])  # True
vis = o3d.visualization.Visualizer()
assert vis.create_window(visible=False)  # no error
```
